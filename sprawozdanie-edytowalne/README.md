# Wersja edytowalna sprawozdania

**Wymóg z dodatku A sprawozdania:**
> *Powinien istnieć oddzielny podkatalog z wersją edytowalną sprawozdania.*

## Co tu trafia

- Główny plik sprawozdania w wersji edytowalnej (`.docx` / `.odt` / `.tex`)
- Wszystkie pomocnicze pliki potrzebne do edycji (style, makra, źródła obrazów wstawionych do sprawozdania)
- Wersje pośrednie (jeśli chcecie zachować historię) — choć Git robi to za was

## Punkty sprawozdania, które tu trafiają

Ten folder zawiera **finalne, scalone sprawozdanie** zawierające wszystkie sekcje:
- 0. Metryczka (uczelnia, wydział, kierunek, skład zespołu, podział pracy, punktacja, adres repozytorium)
- 1. Treść zadania projektowego
- 2. Cel, zakres, kontekst, korzyści *(źródło: TC_p2_5.4)*
- 3. Słownik
- 4. Perspektywa przypadków użycia *(źródło: TC_p4.1, TC_p5.1_5.2)*
- 5. Perspektywa projektowa *(źródło: TC_p5.1_5.2, TC_p4.2_4.3, TC_p2_5.4)*
- 6. Wymagania niefunkcjonalne *(źródło: TC_p5.3)*
- 7. Technologie *(źródło: TC_p5.3)*
- 8. Plan pracy *(źródło: TC_p5.3)*
- 9. Analiza ryzyka *(źródło: TC_p5.3)*
- 10. Kosztorys *(źródło: TC_p5.3)*
- Dodatek A. Przebieg prac zespołu (historia commitów)

## Punkty 0, 1, 3 — zrób tutaj na końcu, gdy reszta jest gotowa

- **0. Metryczka** — uczelnia, wydział, kierunek, przedmiot, rok i semestr studiów, prowadzący, data
  - 0.1 Skład zespołu i podział pracy
  - 0.2 Proponowana punktacja
  - 0.3 Adres publiczny repozytorium GitHub + informacje o dostępie
- **1. Treść zadania projektowego** — wklejcie temat (Programix...)
- **3. Słownik** — definicje specyficznych terminów (np. ONT, switch, rejon obsługi, status zlecenia, pakiet usługi, faktura zbiorcza...)
